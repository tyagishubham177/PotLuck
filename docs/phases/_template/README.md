# Phase Template

## Objective
State the narrow business and engineering goal for the phase.

## Background
Explain why the phase exists now and what earlier work it depends on.

## Prerequisites
- List the docs, phases, and runtime assumptions that must already be true.

## Touched Packages
- List expected packages and apps.

## Explicit Non-Goals
- State what this phase must not try to solve.

## Exit Criteria
- Define the smallest outcome that makes the phase complete.
