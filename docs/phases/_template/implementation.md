# Implementation Template

## Sequence
1. List work in the order it should be implemented.
2. Call out package boundaries and persistence boundaries.
3. Keep later phases out unless a seam is required.

## Keys and Inputs
- State whether this phase needs any new keys, secrets, or external account setup from the user.
- For each item, say where it comes from, where the real value should be stored, and what example file already holds the dummy placeholder.
- If no new key is needed, say that explicitly so the phase can start without waiting.

## Notes
- Identify exact files and modules once the scaffold exists.
- Keep work small enough to validate before moving on.
