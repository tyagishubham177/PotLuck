# Phase 04 UI References

## Purpose
This phase is backend-heavy, but players still need clear, trustworthy realtime state feedback. Use this reference for disconnect and reconnect handling.

## Implementation Notes
- Reconnect states should explain impact on the current hand in plain language.
- Alerts should feel restrained and calm; do not jump to error red unless the state is truly unrecoverable.
- Banners and overlays should use glass treatment so the table context remains visible underneath.
